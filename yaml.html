YAML

YAML é um acrônimo recursivo em inglês para YAML Ain't Markup Language, que em tradução livre significa: YAML não é uma linguagem de marcação. Ansible utiliza este formato devido a sua relativa facilidade de compreensão.


YAML Básico

Para o ansible, todo arquivo YAML deve começar com uma lista. Cada item na lista é também um lista de pares chave/valor que também é conhecido como hash ou dicionário. Para utilizar ansible da forma correta precisamos saber como escrever listas e dicionários em YAML. Todos os arquivos YAML (quer estejam relacionados com o ansible ou não) podem opcionalmente começar com --- e terminar com ... Este conjunto de caracteres indica o começo e o fim (respectivamente) de um documento.

Todos os membros de uma lista devem começar começar numa nova linha e ter o mesmo nível de indentação, por exemplo: 

              ---
              fruits:
                - Apple
                - Orange
                - Strawberry
                - Mango
              ...

Um dicionário é representado simplesmente por um par chave: valor. É necessário um espaço após os dois pontos. 

              martin:
                name: Martin D'vloper
                job: Developer
                skill: Elite

Estruturas mais complicadas também são possíveis, como listas de dicionários, dicionários que possuem listas como valores entre outras: 

              -  martin:
                name: Martin D'vloper
                job: Developer
                skills:
                  - python
                  - perl
                  - pascal
              -  tabitha:
                name: Tabitha Bitumen
                job: Developer
                skills:
                  - lisp
                  - fortran
                  - erlang

Dicionários e listas também podem ser representados de forma abreviada: 

              ---
              martin: {name: Martin D'vloper, job: Developer, skill: Elite}
              fruits: ['Apple', 'Orange', 'Strawberry', 'Mango']
