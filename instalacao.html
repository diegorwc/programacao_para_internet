Instalação

Como instalar o ansible

Por padrão ansible utiliza o protococlo SSH para gerenciar as máquinas. Uma vez que o ansible estiver instalado, não será necessário adicionar um banco de dados e também não é preciso iniciar daemons. Você precisa instalá-lo em apenas uma máquina e pode gerenciar diversos computadores remotamente através deste servidor central. Quando ansible está gerenciando máquinas remotas ele não deixa nenhum tipo de software instalado ou rodando, então não é preciso atualizá-lo no cliente quando você realizar um upgrade no servidor.

Requisitos necessários

Atualmente ansible pode rodar em qualquer host com python 2 (versões 2.6 ou 2.7) ou python 3 (versão 3.5 ou acima) instalado, porém não é possível usar um computador com windows como servidor. Red Hat, Debian, CentOS, OS X e qualquer sistema operacional BSD (entre outros) são compatíveis com ansible.

Requerimentos para os clientes

Nos nós gerenciados, você precisa de uma maneira de se comunicar, que normalmente é o ssh. Por padrão, isso usa o sftp. Se isso não estiver disponível, você pode alternar para scp em ansible.cfg. Você também precisa do Python 2 (versão 2.6 ou posterior) ou do Python 3 (versão 3.5 ou posterior).

Nota
Se você tiver o SELinux habilitado em computadores remotos, você deverá instalar libselinux-python neles antes de usar qualquer função relacionada a cópia / arquivo / modelo no Ansible. Você pode usar o módulo yum ou o módulo dnf no Ansible para instalar este pacote em hosts que não possuirem. Por padrão, o Ansible usa o interpretador python localizado em /usr/bin/python para executar seus módulos. No entanto, algumas distribuições Linux podem ter apenas um interpretador do Python 3 instalado em /usr/bin/python3 por padrão. Nesses sistemas, você pode ver um erro como:

"module_stdout": "/bin/sh: /usr/bin/python: No such file or directory\r\n"

você pode definir a variável de inventário ansible_python_interpreter para apontar para o executável de acordo com seu sistema ou instalar um interpretador do Python 2. Você ainda precisará definir ansible_python_interpreter se o interpretador do Python 2 não estiver instalado em /usr/bin/python. O módulo "bruto" da Ansible (para executar comandos de maneira rápida) e o módulo de script nem precisam do Python instalado. Então, tecnicamente, você pode usar o Ansible para instalar uma versão compatível do Python usando o módulo raw, que permite usar todo o resto. Por exemplo, se você precisar fazer o bootstrap do Python 2 em um sistema baseado em RHEL, você pode instalá-lo via:

ansible myhost --sudo -m raw -a "yum install -y python2"
